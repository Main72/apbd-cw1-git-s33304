1. Kiedy Git wykona fast-forward, a kiedy powstaje merge commit?
1)Git wykonuje ten typ scalenia gdy gałąź docelowa nie zmieniła się (nie otrzymała żadnych nowych commitów) od momentu utworzenia na niej twojej gałęzi roboczej.
Powstaje, gdy historie obu gałęzi się rozeszły 
2. Czym w praktyce różni się merge od rebase?
2)Merge: Zachowuje w 100% oryginalną, historyczną prawdę o tym, jak przebiegała praca.
Rebase: Przepisuje historię projektu, aby była w 100% liniowa i czysta
